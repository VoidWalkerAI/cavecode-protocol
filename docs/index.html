What is CaveCode?

CaveCode is a plain-text format for describing systems: games, tools, teaching materials, configuration files, narrative flows, and more. It is not a programming language. It sits beside the code you already have and makes it legible.

A CaveCode file is organized into labeled BLOCKS. Each block groups related ideas or parameters and carries at least one glyph to signal its role. The file can be read and edited on any device, including a small phone screen.

Plain Text

UTF-8 text files you can open anywhere. No special editor required.

Block-Mapped

Clear sections labeled as BLOCK 1, BLOCK 2, etc., so humans and AI agents can both see the map.

Glyph-Marked

Emojis act as semantic markers. You always know what is safe to touch and what is dangerous to break.

Culture-Friendly

Space for story, philosophy, and teaching notes in the same document as the technical description.

Why CaveCode?

Most code bases are written for machines and experts. CaveCode is written for everyone else:

  • A person on a phone can tweak difficulty, messages, and settings without touching engine code.
  • AI agents can generate, refactor, and validate files without guessing where the important knobs live.
  • Teachers and designers can share artifacts that read like design docs instead of tangled source files.
  • Studios can standardize on a single format for specs, configs, and public documentation.

CaveCode is meant to be small, durable, and remixable: a protocol you can still read in twenty years.

Tools & Badges

Validator (Reference Tool)

A simple Python script lives under /tools/validator/validate_cavecode.py. It performs heuristic checks on CaveCode artifacts:

  • Detects BLOCK headers
  • Checks for standard glyph usage
  • Warns about obviously empty blocks

It is intentionally small and easy to read, so you can fork it and adapt it to your own workflow.

Philosophy

CaveCode is built on a simple belief: the bottleneck is not engineering, it is communication.

A rancher on a hillside with a phone, a student in a crowded lab, a designer in a noisy caf√©, and an AI in a datacenter should all be able to look at the same file and understand the shape of a system.

That is what CaveCode is for:

  • A shared map between humans and machines.
  • A safe space to tweak, learn, and teach.
  • A protocol that can live longer than any single framework.

If you build something with CaveCode, you are invited to add it to the public library, share your patterns, and help grow the next thousand artifacts.
